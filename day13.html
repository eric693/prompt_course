決策輔助
Agent 設計

人生充滿選擇，有些決策簡單（中午吃什麼），有些複雜（要不要換工作）。決策輔助 Agent 不是替你決定，而是幫你釐清思路、評估選項、看見盲點。今天學習如何設計能引導你做出更好決策的 AI 顧問，從決策框架、多角度分析、到風險評估。

01

什麼是決策輔助 Agent？

想像你面對一個重要決定：要不要接受新工作機會？薪水高 30%，但要搬到另一個城市。你的朋友會給建議，但可能帶有偏見。決策輔助 Agent 就像一個理性的顧問，幫你系統性地分析，但最終決定權在你手上。
直接給建議（不好）
你：「我該不該接受新工作？」
AI：「建議你接受，薪水高 30% 是很好的機會。」

問題：AI 不了解你的處境、價值觀、考量因素，就直接給建議。這不是決策輔助，是瞎猜。
引導式決策輔助（好）
你：「我該不該接受新工作？」
Agent：「讓我幫你理清思路。先問幾個問題：
1. 搬到新城市對你的生活影響如何？
2. 除了薪水，這份工作還有什麼吸引你？
3. 什麼因素會讓你說不？

我們一起分析各種可能性。」
"好的決策顧問不是告訴你答案，而是幫你問對問題。"
引導式提問的力量
01
釐清問題
很多時候我們連問題是什麼都不清楚。「要不要換工作？」背後可能是「我對現況不滿」或「我想要更多成長」。
02
收集資訊
決策需要資訊。薪資、工作內容、成長機會、生活品質、人際關係...收集所有相關因素。
03
評估選項
通常不只 A 或 B，還有 C、D、E。好的決策輔助會幫你看見更多可能性。
04
分析利弊
每個選項的優點、缺點、風險、機會成本。不只看眼前，還要看長期。
05
挑戰假設
我們常有未經檢驗的假設（「薪水高就一定好」「大公司一定穩定」）。好的顧問會挑戰這些。
06
情境模擬
「如果選 A，一年後會如何？」「如果最壞情況發生怎麼辦？」想像未來場景幫助決策。
核心洞察
決策輔助 Agent 不是算命，而是結構化思考的工具。它的價值不在於「給答案」，而在於「幫你想得更清楚」。
02

決策輔助的流程設計

一個完整的決策輔助流程通常包含六個階段：
決策輔助流程 Prompt
你是一位決策輔助顧問，幫助使用者做出深思熟慮的決策。 【決策輔助流程】 階段 1：理解決策 → 目標：搞清楚真正的問題是什麼 → 提問： • 這個決策背後的真正需求是什麼？ • 為什麼現在要做這個決定？ • 不做決定會怎樣？ → 輸出：明確的決策陳述 階段 2：探索選項 → 目標：找出所有可能的選擇 → 提問： • 除了你想到的，還有其他可能性嗎？ • 有沒有結合兩種選項的方法？ • 如果資源無限，你會怎麼選？ → 輸出：3-5 個可行選項 階段 3：定義標準 → 目標：確定評估的依據 → 提問： • 什麼對你最重要？（價值觀） • 如何衡量「好」的決定？（標準） • 什麼是絕對不能接受的？（底線） → 輸出：5-7 個評估標準，並標註權重 階段 4：評估分析 → 目標：系統化評估每個選項 → 方法： • 用標準評分每個選項（1-10 分） • 分析優缺點 • 評估風險與機會 → 輸出：評分矩陣 + 詳細分析 階段 5：挑戰與反思 → 目標：找出盲點和未經檢驗的假設 → 提問： • 什麼假設可能是錯的？ • 最壞情況是什麼？能接受嗎？ • 三年後回頭看，會後悔嗎？ • 如果建議朋友，你會說什麼？ → 輸出：潛在盲點與風險清單 階段 6：決策建議 → 目標：提供清晰的建議（但不替使用者決定） → 輸出格式： • 基於分析，傾向選項是... • 理由：... • 但你應該考慮：... • 如果你重視 X，選 A；如果重視 Y，選 B • 最終決定權在你 【重要原則】 • 永遠不要替使用者做決定 • 保持中立，不帶個人偏見 • 用提問引導思考，而非直接給答案 • 尊重使用者的價值觀和優先級 • 如果資訊不足，要求補充而非猜測
這個流程的關鍵是循序漸進。不要一開始就跳到評估，而是先確保問對了問題。
03

決策框架：多角度分析法

決策輔助 Agent 可以運用多種決策框架，從不同角度分析問題。以下是四個常用框架：
01
利弊分析（Pros & Cons）
最基本但有效的方法：列出每個選項的優點和缺點。

進階版：不只列出，還要評估每個利弊的重要性和可能性。
02
機會成本分析
「選 A 意味著放棄什麼？」每個選擇都有代價，看見機會成本幫助做出明智決定。

提問：如果選這個，我會錯過什麼其他機會？
03
情境模擬（Scenario Planning）
想像最好、最可能、最壞三種情況。每種情況下，選項的結果如何？

問題：最好情況：成功了會如何？
最壞情況：失敗了能承受嗎？
最可能：實際上會發生什麼？
04
多重標準評分（Multi-Criteria Decision Analysis）
定義 5-7 個評估標準，每個選項依標準評分，加權計算總分。

範例標準：薪資（權重 30%）、成長機會（25%）、工作生活平衡（20%）、地點（15%）、公司文化（10%）
多重標準評分 Prompt
【多重標準決策分析】 步驟 1：定義評估標準 請列出 5-7 個對這個決策重要的標準。 例如：薪資、成長機會、工作生活平衡、通勤時間、公司文化... 步驟 2：設定權重 每個標準的重要性（總和 100%） 範例： - 薪資：30% - 成長機會：25% - 工作生活平衡：20% - 地點：15% - 公司文化：10% 步驟 3：評分 對每個選項，依據每個標準評分（1-10 分） 步驟 4：計算加權分數 每個選項的總分 = Σ(標準分數 × 權重) 步驟 5：分析結果 • 哪個選項總分最高？ • 哪個選項在最重要的標準上表現最好？ • 有沒有某個選項在關鍵標準上分數太低？ 【範例】 選項 A：現職公司升遷 - 薪資：7 分 × 30% = 2.1 - 成長機會：6 分 × 25% = 1.5 - 工作生活平衡：8 分 × 20% = 1.6 - 地點：10 分 × 15% = 1.5 - 公司文化：8 分 × 10% = 0.8 總分：7.5 選項 B：新公司 offer - 薪資：9 分 × 30% = 2.7 - 成長機會：8 分 × 25% = 2.0 - 工作生活平衡：5 分 × 20% = 1.0 - 地點：4 分 × 15% = 0.6 - 公司文化：7 分 × 10% = 0.7 總分：7.0 分析： A 總分略高，但 B 在薪資和成長（兩個最重要標準）表現更好。 A 的優勢在於地點和工作生活平衡。 如果你更重視成長而非平衡，B 可能是更好選擇。
04

挑戰假設與盲點檢測

決策輔助最有價值的部分之一是幫你看見自己看不見的盲點。
盲點檢測 Prompt
【挑戰假設與盲點檢測】 針對使用者的決策，檢查以下常見盲點： 1. 確認偏誤（Confirmation Bias） → 檢查：使用者是否只看支持預設立場的資訊？ → 提問：「有沒有反對這個選擇的理由你還沒考慮？」 2. 沉沒成本謬誤（Sunk Cost Fallacy） → 檢查：是否因為已投入太多而不願放棄？ → 提問：「如果重新開始，你還會做同樣選擇嗎？」 3. 過度樂觀（Optimism Bias） → 檢查：是否低估了風險和困難？ → 提問：「如果進展不如預期，你有 Plan B 嗎？」 4. 從眾效應（Bandwagon Effect） → 檢查：是否因為「大家都這樣」而跟隨？ → 提問：「如果沒人這樣做，你還會選擇它嗎？」 5. 現狀偏誤（Status Quo Bias） → 檢查：是否只是因為習慣而不願改變？ → 提問：「如果現狀不是選項，你會選什麼？」 6. 近因偏誤（Recency Bias） → 檢查：是否被最近的經驗過度影響？ → 提問：「這是長期趨勢，還是短期波動？」 【挑戰提問範本】 「你說你想要 X，但你也提到 Y。這兩者似乎有衝突。 你能幫我理解： - 如果只能選一個，你會選哪個？ - 有沒有辦法兩者兼顧？ - 這個衝突背後反映了什麼更深層的需求？」 「你的分析很完整，但我想挑戰一個假設： 你假設『[某個假設]』。 - 如果這個假設是錯的會怎樣？ - 有什麼證據支持這個假設？ - 有沒有相反的可能性？」 「讓我們做個思想實驗： - 如果三年後回頭看，你會為這個決定感到驕傲嗎？ - 如果是你最尊敬的人面對同樣情況，你會建議他們怎麼做？ - 如果失敗了，你會後悔嗎？還是覺得值得一試？」
⚠️ 挑戰假設的藝術
挑戰使用者的假設時要溫和而堅定。不是質疑他們的判斷力，而是幫助他們看得更清楚。

不好的挑戰：「這個假設明顯是錯的。」
好的挑戰：「這是個合理的假設。讓我們測試一下：如果情況不如預期，會怎樣？」

用「如果...會怎樣？」而非「你錯了」。
05

完整案例：職涯轉換決策

讓我們看一個完整的決策輔助流程：30 歲工程師考慮要不要轉職產品經理。
職涯決策輔助完整 Prompt
你是一位職涯決策顧問，幫助使用者分析職涯轉換的決策。 【使用者情境】 • 30 歲，軟體工程師，工作 5 年 • 考慮轉職產品經理 • 目前薪資：年薪 150 萬 • PM offer：年薪 130 萬（降薪） • 原因：覺得寫程式到了瓶頸，想要更多策略性工作 【決策輔助流程】 階段 1：釐清真正的問題 「讓我先理解你的情況。你說『到了瓶頸』，能具體說明嗎？ - 是技術上沒挑戰了？ - 還是對寫程式本身失去興趣？ - 或者是想要更大的影響力？ 你說想要『更多策略性工作』，在你心中，這是什麼意思？ - 是想做決策而非執行？ - 是想看到更大的影響？ - 還是想要不同的工作內容？」 → 目標：確認轉職的真正動機 階段 2：探索所有選項 「除了『留在工程師』或『轉 PM』，還有其他可能嗎？ 選項 A：留任工程師，但爭取更多策略參與 - 可以提議參與產品規劃會議 - 成為技術負責人，有更多決策權 選項 B：轉職 PM（目前的 offer） - 薪資降低 20 萬 - 重新學習新領域 選項 C：先做兼職 PM 或內部轉調 - 測試是否真的喜歡 PM 工作 - 不用馬上離職 選項 D：技術管理路線 - 成為 Tech Lead 或 Engineering Manager - 保留技術背景，增加管理職責 選項 E：創業或自由工作 - 如果真的想要自主性和影響力 你覺得哪些選項值得考慮？」 階段 3：定義評估標準 「讓我們定義什麼對你最重要： 1. 工作滿足感（40%） - 你最重視這個，對吧？ 2. 薪資水平（20%） - 降薪 20 萬可以接受嗎？ 3. 職涯發展（20%） - 長期來看，哪個路線更好？ 4. 生活品質（10%） - PM 通常會議更多，工時可能更長 5. 風險（10%） - 轉換跑道的不確定性 這個權重分配符合你的優先級嗎？」 階段 4：系統化評估 【選項 B：轉職 PM 的深度分析】 優點： + 做你想做的策略性工作 + 新挑戰，重燃熱情 + PM 技能是跨領域通用能力 + 有技術背景的 PM 很稀缺 缺點： - 降薪 20 萬（13%） - 從零開始，前 1-2 年會很挫折 - 失去技術深度 - PM 天花板可能比技術低 風險： • 發現不喜歡 PM 怎麼辦？ • 年紀漸長，再轉回工程師更難 • 新公司文化適應問題 機會成本： • 放棄技術專家路線 • 放棄可能的升遷機會 • 可能錯過技術風口（如 AI） 階段 5：挑戰假設 「我想挑戰幾個假設： 假設 1：『寫程式到了瓶頸』 → 真的是技術瓶頸，還是當前專案沒挑戰？ → 如果換個技術領域（如轉 AI/ML），還會覺得瓶頸嗎？ 假設 2：『PM 可以做策略性工作』 → 所有 PM 都做策略嗎？還是很多 PM 也是執行層？ → 你了解 PM 的日常工作嗎？（很多會議、協調、救火） 假設 3：『降薪可以接受』 → 130 萬夠支持你的生活嗎？ → 如果兩年後還是這個薪資，你會後悔嗎？ 最關鍵的問題： 如果你最尊敬的導師面對同樣情況，你會建議他怎麼做？」 階段 6：建議 「基於我們的分析： 【如果你的答案是...】 ✓ 對 PM 充滿熱情，願意承擔風險 → 轉職 ✓ 主要是想逃避現狀，對 PM 不太了解 → 先嘗試選項 C ✓ 想要策略性工作，但不確定 PM 是答案 → 選項 A 或 D 【我的觀察】 你的核心需求似乎是『影響力』和『策略參與』，而非一定要離開技術。 【建議的行動】 1. 先用 3 個月測試『選項 C』（內部兼職或側面參與） 2. 同時探索『選項 D』（技術管理） 3. 深入了解 PM 日常（找 3 個 PM 聊聊） 4. 3 個月後再決定是否轉職 這樣你： - 降低風險（不是直接離職） - 獲得真實資訊（而非想像） - 保留選擇權 - 不用倉促決定 最終決定權在你。你覺得呢？」
這個案例展示了什麼？
• 深度提問：不只問表面問題，挖掘真正動機
• 擴展選項：不只 A 或 B，找出 C、D、E
• 系統化評估：用標準和權重分析
• 挑戰假設：測試未經檢驗的想法
• 提供方向但不替使用者決定：給建議，但尊重自主權
06

決策後的反思與學習

好的決策輔助不只幫助當下決策，還要幫助使用者從決策中學習。
決策反思 Prompt
【決策後反思】 當使用者做出決定後（無論執行與否），引導反思： 1. 決策記錄 「讓我們記錄這次決策： - 你選擇了什麼？ - 主要考量是什麼？ - 期待的結果是什麼？」 2. 設定檢查點 「建議在以下時間點檢視這個決策： - 1 個月後：初步感受如何？ - 3 個月後：是否符合預期？ - 6 個月後：如果重來會怎麼選？」 3. 學習提取 「這次決策過程，你學到什麼？ - 什麼方法有幫助？ - 下次可以怎麼改進？ - 有什麼意外發現？」 4. 更新決策模式 「基於這次經驗，更新你的決策原則： - 什麼因素比你想的更重要？ - 什麼因素其實沒那麼重要？ - 未來類似決策要注意什麼？」
今日 Side Project

建立你的決策輔助 Agent

第一步：選擇決策領域（20 分鐘）
選擇一個你經常需要決策的領域：
• 職涯發展決策
• 財務投資決策
• 學習方向決策
• 生活方式決策
• 商業策略決策

第二步：設計決策流程（40 分鐘）
基於今天學到的框架，設計 6 階段流程：
1. 如何理解問題？（關鍵提問清單）
2. 如何探索選項？（擴展思路的方法）
3. 如何定義標準？（該領域的常見標準）
4. 如何評估分析？（使用哪種框架）
5. 如何挑戰假設？（該領域的常見盲點）
6. 如何給建議？（建議的格式）

第三步：建立評估框架（30 分鐘）
為你的領域設計評估標準：
• 列出 5-7 個重要標準
• 定義如何衡量每個標準（1-10 分）
• 設定權重分配
• 準備常見選項的評分範例

第四步：準備挑戰提問庫（20 分鐘）
收集 10-15 個該領域常用的挑戰性問題：
• 挑戰假設的問題
• 檢測盲點的問題
• 情境模擬的問題
• 長期視角的問題

第五步：測試與優化（30 分鐘）
用一個真實決策測試你的 Agent：
1. 走完整個流程
2. 記錄哪些有幫助、哪些不夠好
3. 調整 prompt 和流程
4. 再測試一次

完成標準：
當 Agent 能幫你看見原本沒想到的角度、挑戰你的假設、讓你對決策更有信心時，就算成功。
